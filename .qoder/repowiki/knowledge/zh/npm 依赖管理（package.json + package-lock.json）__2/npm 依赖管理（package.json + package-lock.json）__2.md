本项目采用标准的 npm 单仓库依赖管理模式，通过 `package.json` 声明依赖、`package-lock.json` 锁定版本，配合 `node_modules/` 本地安装目录完成依赖解析与安装。

**核心文件与工具**
- `package.json`：项目元信息、脚本命令、运行时依赖（`dependencies`）与开发依赖（`devDependencies`）的统一声明入口。
- `package-lock.json`：lockfileVersion 3，记录每个包精确版本、resolved URL 与 integrity hash，确保多环境构建一致。
- `.gitignore`：忽略 `node_modules/` 与 `dist/`，只提交源码与 lockfile。
- README.md 中仅使用 `npm install` 作为安装指引，未引入 pnpm/yarn。

**依赖分组与范围**
- 运行时依赖集中在 UI 组件生态（Radix UI 全量子包、shadcn/ui 衍生库）、表单与校验（react-hook-form + zod）、图表（recharts）、动画（tailwindcss-animate、tw-animate-css）、图标（lucide-react）等。
- 开发依赖围绕 Vite 7 + React 19 构建链，以及 ESLint 9 + TypeScript 5.9 代码质量栈。
- 所有依赖均使用 `^` 或 `~` 语义化版本范围，未固定到具体 patch 版本；实际锁定由 lockfile 保证。

**安装与更新策略**
- 默认使用 npm v8+（lockfileVersion 3），无私有 registry 配置，全部从 npmjs.org 拉取。
- 未发现 `pnpm-lock.yaml`、`yarn.lock`、`.npmrc`、`bun.lock` 等其他锁文件或镜像源配置，也未见 `vendor/` 或 `__vendor__` 等 vendoring 目录。
- 升级依赖需手动编辑 `package.json` 后运行 `npm install` 生成新的 lockfile，无自动化更新脚本（如 dependabot、renovate）可见于仓库根目录。

**开发者约定**
- 新增依赖应写入 `package.json` 对应字段，并提交同步更新的 `package-lock.json`。
- 禁止将 `node_modules/` 提交至版本库。
- 保持 `type: "module"` 的 ESM 模式，避免混用 CommonJS 引入。
