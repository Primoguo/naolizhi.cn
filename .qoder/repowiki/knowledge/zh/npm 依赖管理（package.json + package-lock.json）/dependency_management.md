本项目采用 npm 作为包管理器，通过根目录的 `package.json` 声明依赖，配合 `package-lock.json`（lockfileVersion: 3）锁定所有子依赖版本，确保构建可重复。项目标记为私有（`"private": true`），不发布到 npm 仓库。

- **依赖范围**：全部第三方依赖集中在根级 `dependencies` / `devDependencies` 下，无 monorepo、无 workspace、无子包拆分。
- **版本策略**：生产与开发依赖均使用 `^` 前缀（如 `react ^19.2.0`、`vite ^7.2.4`），允许 npm 在语义兼容范围内升级；TypeScript 使用 `~` 精确到次版本（`~5.9.3`）。未使用固定版本号或 Git 源。
- **锁文件**：`package-lock.json` 由 npm 自动生成与维护，提交至版本库，用于 CI/CD 与团队协作时复现完全一致的依赖树。
- **私有源/代理**：未发现 `.npmrc`、`.yarnrc`、`pnpm-workspace.yaml` 等自定义注册表配置，也未见 `GOPRIVATE` 或 Go 模块相关文件，表明仅使用默认 npm 公共源。
- **Vendoring**：依赖安装到 `node_modules/`，未启用任何 vendoring 机制（如 `--frozen-lockfile` 之外的本地拷贝策略）。
- **脚本约定**：`build` 先执行 `tsc -b` 再 `vite build`，将类型检查纳入构建流程；`lint` 基于 ESLint 9 + TypeScript ESLint。

开发者应遵循的约定：
1. 新增依赖统一写入根 `package.json`，区分 `dependencies` 与 `devDependencies`。
2. 优先使用 `^` 保持小版本自动更新，对编译器类工具（如 TypeScript）用 `~` 收紧范围。
3. 修改 `package.json` 后必须重新生成并提交 `package-lock.json`，禁止手动编辑 lock 文件。
4. 不使用私有 npm 源或 vendor 目录，保持依赖来源透明可审计。
