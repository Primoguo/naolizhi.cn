单页入口 `src/main.tsx` 通过 `createRoot` 挂载 `<App />`；`src/App.tsx` 作为页面装配器，自上而下组合 `sections/` 下的全屏区块组件（Navbar、Hero、Features、TTSDemo、Highlights、CTA、Footer），并在最底层以固定定位叠加 `FluidCanvas` 与 `EarParticles` 两个全屏渲染层。

- 目录分层：`components/ui` 为 shadcn/ui 生成的原子 UI 组件，`components/` 存放业务级可视化组件（AudioWaveRing、BlackHoleVisualizer 等），`hooks/` 提供可复用交互钩子（magnetic、scroll-reveal、spotlight、reduced-motion 等），`lib/` 封装 WebGPU 设备探测与 FBO/Uniform 缓冲创建等底层 API（`webgpu.ts`、`webgpu-renderer.ts`）。
- 构建与路径：`vite.config.ts` 配置 `@` 别名指向 `./src`，`base: './'` 适配 GitHub Pages 相对部署；`tsconfig.app.json` 配合 `tsc -b` 在 `build` 脚本中先类型检查再打包。
- 样式体系：`tailwind.config.js` 启用 `darkMode: 'class'`，通过 CSS 变量映射 hsl 色板，并注册 `tailwindcss-animate` 与自定义 keyframes；shadcn 通过 `components.json` 声明 `new-york` 风格、`lucide` 图标库及 `@/components`、`@/hooks` 等别名。
- 渲染管线：`FluidCanvas` 内嵌完整 WebGL 流体模拟（splat/advection/divergence/pressure/gradientSubtract/display 着色器阶段），并通过 `IntersectionObserver` + FPS 自适应降级控制性能；`lib/webgpu.ts` 提供设备探测、FBO/Uniform 缓冲创建等底层 API，供其他 WebGPU 组件复用。
